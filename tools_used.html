QGis

QGIS was employed as the main platform for the processing, analysis and visualization of spatial data. Multi-year pollutant maps (NO₂, PM₂.₅), land cover datasets and population distributions were imported, reprojected, and clipped to the national extent of Kosovo. Pollution layers were processed using the “Raster Calculator” to compute the Annual Average Difference and then reclassified according to EU health thresholds. Population data were divided into quantile-based classes to ensure comparability.

“Zonal Statistics” were applied to extract the maximum pollution class and median population class within administrative areas. These were combined into a single bivariate index to produce an exposure map, highlighting zones of environmental concern. Additional tools such as “Dissolve” and “DataPlotly” were used to aggregate population statistics and generate graphical summaries, including pie charts, which were later integrated into the project website.

Visual Studio Code

Visual Studio Code was adopted as the primary development environment for the implementation of the project website and the integration of the WebGIS component. Its versatility and compatibility with standard web technologies such as HTML, CSS, and JavaScript made it particularly suitable for structuring and customizing the website. The website was designed to include multiple sections, aimed at communicating the objectives, methodology, datasets, and outcomes of the project. HTML templates were customized and extended directly within the Visual Studio Code environment to ensure clarity, consistency, and responsiveness.

JavaScript libraries, particularly OpenLayers, were incorporated to implement the interactive web map. Functionalities such as the scale line, full screen toggle, mouse position display, and layer switching were integrated using code editing and tested directly within the VSC interface. The editor also facilitated real-time preview, syntax checking, and effective project file organization, thereby supporting a streamlined and efficient development process.

Geoserver and FileZilla

GeoServer is an open-source server software designed to facilitate the sharing, editing, and publication of geospatial data through standard web services such as WMS and WFS. Within the framework of this project, GeoServer was used to publish the layers generated in QGIS, in order to make them accessible and usable directly in the WebGIS interface.

The layers were first prepared and styled in QGIS, then exported (as GeoTIFF or GeoPackage files) and uploaded to the designated GeoServer instance. This transfer was carried out using FileZilla, a secure FTP client, which allowed the upload of the raster and vector datasets into the appropriate directory on the server. Once uploaded, the layers were configured within the GeoServer interface, where the appropriate coordinate reference system, styling (SLD), and service settings were defined.

These published layers were then integrated into the website using OpenLayers, enabling interactive visualization of spatial data directly from the server through standard WMS requests. This workflow ensured centralized data management, dynamic rendering, and efficient delivery of spatial information in the final web platform.

Excel

Microsoft Excel is a widely used spreadsheet application designed for data organization, analysis, and visualization. In the context of this project, Excel was utilized to generate simple charts and graphs based on extracted statistical data, supporting the interpretation of population exposure and pollutant distributions.
